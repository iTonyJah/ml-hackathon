## Отредактированные существующие документы

### `README.md`

Что изменено:

- Документ переписан с baseline-описания на описание финального решения.
- Добавлены ссылки на новый основной маршрут документации.
- Добавлены команды подготовки validation split.
- Добавлены команды eval для двух сценариев:
  - локальный split;
  - готовый train/validation набор.
- Добавлена команда time-based CV.
- Обновлен раздел "Где что лежит" под финальную архитектуру:
  - `ml_model.py`;
  - `prepare_manager.py`;
  - `cv.py`;
  - split-скрипты в `scripts/`.
- Добавлено краткое объяснение целевой метрики.

### `HOW-TO.md`

Что изменено:

- Документ синхронизирован с финальным ML-пайплайном.
- Добавлены команды:
  - `scripts/create_validation_split.py`;
  - `scripts/create_validation_split_2d.py`;
  - eval для локального split;
  - eval для готового validation;
  - time-based CV.
- Добавлено описание, где менять решение.
- Добавлено простое объяснение, как работает финальный `predict`.
- Обновлен раздел частых проблем:
  - сервис не запущен;
  - порт занят;
  - `predict` возвращает `503`;
  - eval дает нули.

### `TRAIN.md`

Что изменено:

- Документ разделен на два смысла обучения:
  - legacy/baseline train pipeline;
  - финальный online-путь через `PrepareManager` и `MLModel`.
- Добавлено описание target construction:
  - positive через `APPLY` до `shift.start_at`;
  - negative через `VIEW` или `USER_CANCEL` без `APPLY`;
  - исключение `SYSTEM_CANCEL`;
  - фильтрация post-start `APPLY`.
- Добавлены группы признаков модели.
- Добавлено описание `LGBMClassifier` и AUC early stopping.
- Добавлено описание sleeper-rerank.
- Добавлен запуск time-based CV.

### `DATA.md`

Что изменено:

- Документ переписан в более структурированном виде.
- Добавлены таблицы колонок для `user.csv`, `shift.csv`, `event.csv`.
- Добавлено важное пояснение по `validation/apply.csv`:
  - `date` должна быть датой начала смены;
  - `APPLY` после `shift.start_at` не должен попадать в ground truth.
- Добавлено описание файлов, которые появляются после локального split:
  - `shift_train.csv`;
  - `event_train.csv`;
  - validation-файлы.
- Добавлен список проверок корректности данных перед eval.
